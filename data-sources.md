# 데이터 소스 & API 한도 메모

실제 데이터 연동 시 참고. 데모는 `index.html` 내장 모의 데이터로 동작.

## 1차 소스 (공식 · 무료)

| 소스 | 엔드포인트 | 제공 형태 | 한도/주의 |
|------|-----------|----------|-----------|
| 네이버 데이터랩 (통합검색어 트렌드) | `https://openapi.naver.com/v1/datalab/search` | **상대값(기간 내 최댓값=100)**, 일/주/월 단위 | 절대 검색량 미제공 → 증가율은 신뢰, 종목 간 절대 비교는 보정 필요. 그룹 최대 5개, 그룹당 키워드 최대 20개 |
| 네이버 뉴스 검색 | `https://openapi.naver.com/v1/search/news.json` | 기사 메타(제목·링크·날짜) | 일 25,000건 한도 → 키워드 우선순위 큐로 분배 |
| 구글 트렌드 | `pytrends` (비공식 래퍼) | 상대 검색 관심도 | IP 제한 → 호출 간격·배치, 실패 시 캐시 대체 |
| 주가/거래량 | 공공데이터포털 KRX 또는 증권사 API | OHLCV | FOMO 점수와 가격 상관 검증용 |

## 확장 소스 (비공식 · 위험)

| 소스 | 상태 | 비고 |
|------|------|------|
| 디시인사이드 / 에펨코리아 / 블라인드 | 공식 API 없음 | 크롤링은 약관 위반·IP 차단 위험 → **제휴/공개데이터만**, 무단 크롤링 배제 |
| X(트위터) | 유료 API | 월 $100+ → 초기 제외 |
| 네이버 종목토론방 | 비공식 | 안정성 낮음 → 확장 항목 |

## 키 발급

- 네이버: https://developers.naver.com → 애플리케이션 등록 → 데이터랩(검색어 트렌드) + 검색 선택 → Client ID / Client Secret
- 환경변수로 관리 (`.env`), 절대 커밋 금지

## 비용·한도 관리 원칙

1. LLM 요약은 TOP N(예: 상위 20)만 호출 → 호출량 상한 고정
2. 뉴스 API 일 한도 내에서 키워드 우선순위 큐로 분배
3. 모든 collector에 fallback 캐시 → Graceful Degradation
